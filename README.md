# CS2 Case Lab v7

Фан-симулятор открытия кейсов CS2 для GitHub Pages. Проект полностью статический: HTML/CSS/JS, без реальных платежей, Steam-логина, депозитов и вывода предметов.

## Что в v7

- Исправлена логика баланса: списание происходит до старта рулетки, начисление — после продажи/рекламы/колеса.
- Исправлен инвентарь: предметы сохраняются в `localStorage` и отображаются на отдельной странице.
- Добавлен новый ключ сохранения `cs2_case_lab_v7_state`, есть импорт старых v6/v5 сохранений.
- Добавлен сброс старого service worker/cache, чтобы GitHub Pages не отдавал старые `app.js` и `styles.css`.
- Увеличено количество кейсов: официальные кейсы, коллекции CS2/Armory, quality-кейсы, ножи, перчатки, стикеры турниров, агенты, брелоки/charms, нашивки/patches.
- В каждом кейсе настроены скрытые профили окупа/редкости. Они не выводятся в интерфейсе.
- Предметы используют реальные названия и изображения из открытого ByMykel CSGO-API, а при сбое сети включается fallback-пул.
- Цены сделаны в LC как Steam-like market estimate. Для настоящих live-цен Steam нужен backend/API-ключ или подготовленный price-cache.

## Как залить на GitHub Pages

1. Распакуй архив.
2. Загрузи **содержимое папки `cs2-case-lab-v7`** в корень репозитория, а не саму папку.
3. В GitHub: Settings → Pages → Deploy from branch → `main` / root.
4. После обновления сайта нажми `Ctrl + F5`. Если старый PWA уже был установлен, удали и установи заново.

## Локальный запуск

Можно открыть `index.html` напрямую, но для service worker/PWA лучше запускать через локальный сервер:

```bat
cd /d путь\к\cs2-case-lab-v7
python -m http.server 8000
```

Открыть:

```text
http://localhost:8000/index.html
```

## Где менять рекламу своих проектов

В `app.js` найди функцию `projectCards()` и замени ссылки `#` на свои проекты.

## Ограничения GitHub Pages

- Реальный лимит рекламы «10 на IP в сутки» невозможен без сервера. В проекте лимит реализован через `localStorage`, то есть на браузер/устройство.
- Live-цены Steam для всех предметов невозможно стабильно получать с чистого статического сайта из-за CORS/rate-limit/API-ограничений. В коде используется Steam-like оценка и реальные названия/изображения предметов.
